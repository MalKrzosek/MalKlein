About

    Design

    Each of the three Blender models has its own HTML page and shares a common ‘styles.css’ for visual consistency.

    Navigation elements (home button and logos) link back to the landing page.

    The model pages are minimalistic and clean, using the same white, pink, and black colour palette. Originally, each page had separate CSS, but based on feedback from course mates , I unified them to feel more cohesive.

    • The 3D scene is centred inside a .scene div.

    • Three.js renders the models inside an HTML canvas.

      • It took me a lot of time to figure out how to make it work. I started following the tutorials provided on Canvas but my models weren’t appearing. It took me another week, multiple versions of Java Script and help from my course mates to finally make it work!


    Background Images

    To each model I added background images loaded via THREE.Texture loader.

    • Diet Coke background is a sky image that I found online. I colour corrected it to match the sky from the thumbnail animation.

    • Balloon Dog background is the 3D scene I made for the thumbnail animation.

    • The Flower model background is an image that I found online and expanded it using AI generative expand in Photoshop.


    Lightning

    I attempted HDR environment lighting to simulate reflections. Unfortunately every time I tried to load .hdr image using RGBELoader and assign it to the scene, I was failing so I eventually resigned and ended up using Hemisphere, Directional, and Ambient lights with ACESFilmicToneMapping. I still was not happy how my model materials look so I tweaked them to give them some highlights and shiny look using Metalness, Roughness and Environment map intensity.


    After some final improvements, I tweaked the lightning of the Diet Coke model. I used a white light to simulate the sun and added a light with pink hue. I was trying to make it look like a sunset.


    Interactivity

    Users can orbit around every model using mouse controls.

    Each model page has three buttons: two to trigger specific animations, and one to toggle wireframe mode.

    I have the animations to only play once when triggered. One model is hidden and and the other is shown.

    Below the canvas there is a ‘Play next’ section which helps navigate between models.It invites users to stay on my website and keep exploring.

    I Integrated audio to complement the animations. I found the relevant sounds on https://freesound.org/. Audio is triggered through buttons and plays immediately to match the animations.

I really enjoy working in Blender, and since I hadn’t used some of the modelling techniques covered in this process before, I spent a lot of time perfecting the details.

    Challenges

    I exported all my models as glb files. When I first checked how they look in glTF Viewer, they didn’t look right. The animations weren’t playing in sync, some of the elements appeared distorted or disconnected from the models. My flower model had missing textures.

    -It took me another week to analyse what is wrong with the export.

    I found out that all animations have to be Grouped by NLA Tracks. I had to open the NLA Editor and assign ‘New Action’ to every animated element. I also had to make sure that the animation of every element has the same duration so they play in sync.

    Challenges: Baloon Dog

    I was modelling my objects using modifiers so in order for them to be visible in .glb export, I had to make sure they are applied. This unfortunately caused some reposition to my ballon dog model -its legs shifted to one side.- I had to position the elements again.

    Because of the initial export challenges, my transitions between ‘Dog Sitting’ model and ‘Dog Laying Down' are not seamless. As I was repositioning the elements in separate models which made them different.I was planning to change it as it definitely would improve the quality of user experience - the shift during transition between models is very visible- but I had to move over to other elements of my project.

    Figure 8: Side to side comparison of the ending position of model 1 and starting position of model 2.


    While making some improvements, I tried to reposition the legs for better transition. The diffrence is still noticable, but it looks better than before. I also changed the colours of the models to better match with the thumbnail animation.


    Challenges: Flower

    The solution for missing textures in the Flower model was swapping them for less complicated ones. -Again I had to do it separately for the two models so the textures are not seamlessly matching between them which ruins the transition. As I was not expecting to face so many issues, which caused a delay in my planned working schedule I decided to leave it as it is and move over.

    Figure 9: Side to side comparison of materials used for model 1 and model 2.

    After all, I managed to change the mismatching textures to create seamless transition!


    Challenges: Diet Coke

    After fixing the initial animation issues with Diet Coke model, I was hoping for seamless transition on my website…Yet another problem appeared! While it looked great in different .glb viewers, after linking to my website through Java Script, my model appears with missing textures. I was trying various tutorials to fix it but I couldn’t find the issue.

    Figure 10: Diet Coke can model in threejs editor, glTF Viever and loaded on my website.


    While making some final improvements, I decided to try one more time make it look better. I still couldn’t find the issue why the textures were not displaying. The idea was to import the .glb model to blender and try to 'wrap' the label again. After uploading it to my coke.js and checking if it works.

    Figure 11: Diet Coke label test.

    I added cylinder, deleted the caps, resized it to the size of my can. Then I moved to the shading tab and created new material. I added texture node, mapping and texture coordinates. I added my label. I then repositioned it using UV editing. I then entered editing mode and using loopcut and resizing tool tried to match the can shape. I finnished it off by applying the shrink wrap modifier and shading it smooth. It ended up not looking as great as the original model, but it was still a huge improvement from what I previously had. I found out that my can looked so dark because it was very metallic. So to override my GLB model texture settings, I updated the metalness and roughness in the code. This helped to reveal more color and balanced the overall look. I then changed the lightning setup to simulate sunset and add some warmth. I added warm fillight, rimlight, bottom light and soft ambient light.

    unfortunately, my idea with adding the label on cylinder didn't really work for my crush animation.

    Figure 12: Diet Coke model with reapplied label on cylinder, crush animation.


    Testing

    I manually went through every element to check if everything is linked correctly. I checked all buttons to make sure they work. All animations, sounds, interactive elements and pages work as expected, apart from a few known issues.

    I tested the site across different screen sizes and confirmed they work properly. I made sure that my contact details are accurate.

    I did some user testing with my friends. I asked them how they feel about the navigation, interaction with models, sounds and colours. I applied some of the feedback and moved away from various pastel colours of my model pages and consistently applied the same colour palette across the entire site.
